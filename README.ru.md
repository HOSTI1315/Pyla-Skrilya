# Pyla-Skrilya

> [English version](README.md)

Полусамостоятельный форк бота для Brawl Stars, заточенный под режим **Showdown (trio)**. Изначально ответвлён от исходников xxz-форка PylaAI, но значительная часть кода уже своя — собственная веб-панель на FastAPI, обвязка вокруг Brawl Stars API для подгрузки трофеев и Push All, обнаружение застревания по стенам, поддержка эмулятора MuMu и экспериментальные профили производительности.

## Кредитсы (благодарности)

Без этих репозиториев проекта бы не было:

- **PylaAI** — оригинальный бот — https://github.com/PylaAI/PylaAI (авторы: Iyordanov, AngelFire)
- **PylaAi-XXZ** — форк исходников, от которого отделилась эта ветка — https://github.com/xxz-888/PylaAi-XXZ

Всё, что относится к Showdown (аналоговое движение, расстановка по тимке, обход газа, полукруговой манёвр-побег), весь стек `web_ui/` + `backend/`, обвязка вокруг Brawl Stars API и поддержка MuMu — локальные доработки поверх этих апстримов.

> **Важно.** Это сборка для разработчиков. Для стабильной поддерживаемой версии используйте официальный билд PylaAI из их Discord — не этот репозиторий.

## Что есть в этом форке

### Геймплей в Showdown (trio)
- **Аналоговый стик** — непрерывный угол вместо нажатий WASD; плавнее движение и дожжи.
- **Следование за тиммейтом с гистерезисом** — если врага рядом нет, идёт за выбранным союзником и не «прыгает» между двумя соседними.
- **Расстановка в тройке** — не прилипает к тиммейтам, обходит их по орбите, тянется обратно к команде, а не уходит в одиночку далеко.
- **Пассивный рoam** в одиночестве и безопасности.
- **Уход от газа** — определяет массу газа и, если доверенная масса заходит в радиус бегства, перебивает движение и убегает в противоположную сторону.
- **Стенной анти-стак + полукруговой манёвр** — если стены вокруг перестают двигаться, хотя бот командует движение, значит он во что-то упёрся. Бот отступает от препятствия и обходит его дугой. Сторона дуги чередуется между срабатываниями.
- **Учёт трофеев по месту** — распознаёт экраны 1/2/3/4 места.

### Веб-панель (`web_ui/` + `backend/`)
FastAPI-бэкенд раздаёт SPA-панель, заменяющую старый Tkinter-хаб для повседневной работы.

- Дашборд: трофеи в реальном времени, WR, серия побед, очередь целей по каждому браулеру (своя цель на каждого).
- **Push All** с настраиваемым порогом трофеев — через Brawl Stars API ставит общую цель всем браулерам ниже порога.
- Страницы статистики и истории матчей на основе `cfg/match_log.jsonl`.
- Настройки: токен Brawl Stars API / авто-обновление, Discord webhook, профили производительности, язык (EN/RU).

Точки входа бэкенда: [backend/server.py](backend/server.py) (FastAPI + WS), [backend/bot_runner.py](backend/bot_runner.py) (тот же цикл `Main`, что и `pyla_main`), [backend/lobby_scanner.py](backend/lobby_scanner.py) (OCR по экрану выбора браулера). [sessions.py](sessions.py) хранит метаданные сессий.

### Восстановление и контроль
- Перезапускает Brawl Stars, если он закрылся или поверх него выскочило другое окно.
- Жмёт **Reload** в диалоге Idle Disconnect (работает и на светлом оверлее LDPlayer, и на тёмном MuMu).
- При зависании видеопотока перезапускает scrcpy, а не саму игру.
- Окно `Pyla Control` позволяет безопасно ставить движение на паузу и снимать с паузы прямо во время работы.

### Эмуляторы
- LDPlayer (по умолчанию) и **MuMu 12 / старые MuMu** — быстрая 50 мс TCP-проба сократила подключение через ADB с ~54 с до ~2 с и учитывает выбранный эмулятор (запущенный LDPlayer больше не «закрывает» подключение MuMu).

## Установка

В этом форке **нет `setup.exe`** — установка идёт обычным Python-флоу.

1. Установите **Python 3.11 (64-bit)** и Git.
2. Клонируйте репозиторий и зайдите в его папку.
3. Запустите `python setup.py install` — он подтянет все нужные Python-пакеты и подберёт рабочий ONNX Runtime (GPU при возможности, иначе CPU).
4. Запустите Android-эмулятор (LDPlayer или MuMu) и откройте Brawl Stars.
5. Поставьте в эмуляторе разрешение **1920×1080** — это рекомендуемый режим.
6. Запустите `python main.py` (или двойной клик по [run.bat](run.bat)).
7. В веб-панели: выберите эмулятор, настройте браулеров, нажмите **Start**.

### Флаги запуска
- `python main.py` — поднимает веб-панель на `http://127.0.0.1:8765`.
- `python main.py --legacy` — возвращает старый Tk-хаб.
- `--host`, `--port`, `--no-browser` — стандартные серверные флаги.

## Brawl Stars API (Push All + ручной ввод трофеев)

1. Создайте аккаунт разработчика на https://developer.brawlstars.com/.
2. Откройте [cfg/brawl_stars_api.toml](cfg/brawl_stars_api.toml) и заполните:
   ```toml
   player_tag = "#YOURTAG"
   developer_email = "YOUR_DEVELOPER_EMAIL"
   developer_password = "YOUR_DEVELOPER_PASSWORD"
   ```
3. Player tag можно также задать в веб-панели на странице настроек.

Поведение:
- **Push All** (см. ниже) тянет трофеи через API, чтобы решить, кого ставить в очередь.
- **Авто-обновление** логинится в официальный портал разработчика, определяет текущий публичный IP, удаляет старые ключи, созданные PylaAI, создаёт свежий ключ под этот IP и сохраняет токен локально.
- `delete_all_tokens = false` оставляйте как есть — иначе будут стёрты **все** ключи аккаунта разработчика.
- **Никогда** не коммитьте заполненный `brawl_stars_api.toml`. В коммитах токен/почта/пароль должны быть пустыми.

> **Важно.** Авто-заполнение поля «Текущие трофеи» по каждому браулеру **пока работает нестабильно**, особенно при запуске нескольких инстансов — поля могут оставаться пустыми или устаревшими. Считайте, что трофеи у браулеров вы сейчас вводите вручную; на API завязан только Push All, и он работает.

### Push All
- Заранее заполните `cfg/brawl_stars_api.toml`.
- Откройте Brawl Stars на экране лобби.
- Веб-панель: введите целевые трофеи рядом с кнопкой **Push All** на дашборде и нажмите. Бот возьмёт всех браулеров ниже порога (по возрастанию трофеев) с `push_until = цель`.
- Legacy Tk: `Push All 1k` (старый алиас на потолок 1000) или новый диалог с произвольной целью.

## Производительность

> Профили производительности **экспериментальные** — код в [performance_profile.py](performance_profile.py) и CLI работают, но тюнинг до конца не доведён. Воспринимайте их как стартовую точку, а не гарантированный фикс, и проверяйте IPS после применения.

Если бот падает до 1–3 IPS, а Python грузит CPU слабо — попробуйте безопасный профиль захвата и перезапустите:

```
python tools/apply_performance_profile.py --profile balanced
```

`--profile low-end` — для старых ноутбуков, которые троттлят и греются.

### Чек-лист отладки
- Запустите [tools/performance_check.py](tools/performance_check.py).
  - Если выводит `CPUExecutionProvider`, перезапустите `python setup.py install` или поставьте `cpu_or_gpu = "directml"` в `cfg/general_config.toml`.
  - Если `1–2 IPS` при низком CPU, посмотрите строку `scrcpy frame FPS` — низкий FPS значит, что узкое место это эмулятор/ADB, а не модель.
- Ноутбуки с двумя GPU: в настройках графики Windows для `python.exe` и эмулятора поставьте **High performance**.
- DirectML работает, но медленно: попробуйте `directml_device_id = "1"` в `cfg/general_config.toml` и перезапуститесь.
- Отключите режим Efficiency у эмулятора в Windows — он может зажимать кадры и держать бота на 2–5 IPS.
- Для LDPlayer / MuMu выберите соответствующий эмулятор в UI или поставьте `current_emulator = "LDPlayer"` / `"MuMu"` в `cfg/general_config.toml`. Используйте 1920×1080 landscape, FPS эмулятора = 60, отключите low-FPS / eco-режим.
- Держите свободную RAM. Выше ~85% занятости — закройте Discord, браузер и другие игры.
- Включите **Debug Screen** в Additional Settings — получите live-оверлей (player, teammate, enemy, wall, fog, range).

## Замечания

- Это **localhost-сборка**: логин, онлайн-статистика, авто-обновление списка браулеров, иконок и модели стен отключены. Чтобы перевести в онлайн, поменяйте базовый URL API в [utils.py](utils.py) и реализуйте соответствующие эндпоинты.
